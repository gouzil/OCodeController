## 为什么

目前聊天页面每次只加载20条消息，用户向上滚动时无法加载更早的消息。希望查看历史对话内容的用户无法通过 UI 访问这些信息。

## 改动内容

- 在消息列表上添加滚动到顶部检测，当用户到达顶部时触发加载更早的消息
- 将加载的历史消息添加到列表顶部，同时保持当前滚动位置不变
- 加载历史消息时显示加载指示器
- 处理边界情况：无更多历史、加载中、网络错误
- 复用现有的 `loadHistoryPage` 游标分页 API

## 能力

### 新增能力

- `load-more-messages`：支持用户滚动到消息列表顶部时加载更早的消息。当用户到达顶部（或接近顶部）且存在更早消息时，获取上一页并添加到列表头部。加载后保持滚动位置不变。

### 变更能力

- （无）

## 影响范围

- **修改的文件**：`entry/src/main/ets/pages/ChatPage.ets`、`entry/src/main/ets/viewmodel/ChatViewModel.ts`
- **使用的 API**：`ChatViewModel.loadHistoryPage`（现有的游标分页）
- **UI 改动**：在获取历史消息时在消息列表顶部显示加载指示器；调整滚动行为
