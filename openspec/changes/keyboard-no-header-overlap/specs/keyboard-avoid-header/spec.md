## 新增需求

### 需求：标题栏在键盘弹出时保持固定

系统在软键盘弹出时必须保持标题栏固定在屏幕顶部，不随页面内容上移。

#### 场景：键盘弹出
- **当** 用户点击输入框唤起软键盘时
- **则** 根 `Stack` 不再扩展到键盘区域
- **并且** 标题栏保持在 `y=0` 位置，不被推出屏幕

#### 场景：键盘隐藏
- **当** 软键盘隐藏时
- **则** 标题栏仍保持在屏幕顶部
- **并且** 页面恢复正常布局

### 需求：输入框跟随键盘上移

系统在软键盘弹出时必须将输入框区域向上推动，使其不被键盘遮挡。

#### 场景：键盘弹出后输入框可见
- **当** 软键盘弹出时
- **则** 输入框 `Row` 通过 `expandSafeArea([SafeAreaType.KEYBOARD], [SafeAreaEdge.BOTTOM])` 向上调整位置
- **并且** 用户可以正常输入文字

#### 场景：键盘隐藏后输入框恢复
- **当** 软键盘隐藏时
- **则** 输入框 `Row` 恢复到屏幕底部正常位置

### 需求：消息列表在键盘弹出时自动调整

系统在键盘弹出时必须调整消息列表的显示区域。

#### 场景：消息列表自动适配可用高度
- **当** 键盘弹出、输入框上移后
- **则** 消息列表 `Scroll` 的 `layoutWeight(1)` 自动填满剩余空间
- **并且** 用户仍可查看历史消息
