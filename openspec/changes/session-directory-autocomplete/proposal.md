## 为什么

目前新建会话页面的工作目录输入框，只有在用户先点击输入框获得焦点后，才会显示下拉提示。用户在输入文字的过程中不会自动触发目录搜索和下拉选单提示，用户体验不够流畅，目录路径只能靠手动输入，容易出错。

## 改动内容

- 优化目录输入框：当用户在输入框中输入文字时，自动触发目录搜索并显示下拉提示列表
- 获得焦点时，如果输入框已有内容则立即搜索；否则显示最近使用过的路径列表
- 保持现有防抖逻辑（400ms），避免频繁请求
- 当下拉列表展开时，用户点击列表项后自动填充路径并收起列表

## 能力

### 新增能力

- `directory-autocomplete`：在新建会话页面，当用户输入工作目录路径时，自动获取目标后端的文件系统内容并通过下拉框提供选择。输入文字时实时搜索，已有路径时显示最近使用记录。

### 变更能力

- （无）

## 影响范围

- **修改的文件**：`entry/src/main/ets/pages/SessionDetail.ets`
- **API 使用**：`/file`（列出目录）、`/find/file`（搜索目录），均已存在
- **依赖组件**：`PathAutocomplete.ets`（下拉提示 UI）、`SessionDetailViewModel.ts`（搜索逻辑）
